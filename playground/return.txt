この章では、return文を追加して、

a = 3;
b = 5 * 6 -8;
return a + b / 2;

をコンパイルできるようにする。

return文はプログラムの途中に書いても構わないと言うことにする。
通常のCと同様に。プログラムの実行は最初のreturn分で打ち切られて関数からリターンする。

return 5;
return 8;

これは、8を返す。

この機能を実装するため、まずはreturn文を追加した文法がどうなるのか。
今までは、ステートメントはただの式と言うことになっていたが、新たな文法では以下になる。

program = stmt*
stmt    = expr ";" | "return" expr ";"

これを実装するには、トークナイザ、パーサ、コードジェネレータの全てに少しづつ手を加える必要がある。

まずは、トークナイザでreturnyというトークンを認識できるようにする。

TK_RETURNという型のトークンで表す。

return や while, intのように、文法上特別な意味を持つトークン(キーワード)は限られた個数しかないので、このようにトークンごとに別の方を持たせるようにした方が簡単。

文法の階層

入力の何らかの規則に合致しているかどうかを判定するために、「正規表現」というものがよく使われますが、ある程度より複雑な文法は、正規表現で表現することはできません。
例えば、文字列の中でカッコの釣り合いが取れているか判定する正規表現は原理的に書くことができません。

文脈自由文法（BNFで表現できる文法）は正規表現よりも強力で、例えばカッコの釣り合いが取れている文字列だけを表すことができる。
ss = "(" S ")" | ?

しかし、正規表現と同様に文脈自由文法にも限界があり、文脈自由文法では普通のプログラミング言語に出てくる複雑なルールを表現できません。

例えば、「変数は使う前に宣言しなければならない。」というルールはCの文法の一部ですが、こういったルールは文脈自由文法では表現できない。

C言語のコンパイラを書けば、コンパイラにバグがない限り、「コンパイラが受け付けた入力は正しいCプログラムで、受け付けなかった入力は不正なCプログラム」ということができる。
つまり、普通のコンピュータの能力があれば、「Cの文法に一致しているかどうか」という問題は判定可能であり、コンパイラというもの全体として、文脈自由文法より強力な文法判定機だということがでいる。

このように、文法に一致しているかどうかを常にYES/NOで判定できる文法のことを、[Decidable]という。

Decidableではない文法を考えることもできる。

例えば、「コンピュータプログラムが入力として与えられてそれを実行した時、そのプログラムが最終的にexit関数を実行して終了するか、あるいは無限に実行を続けるかどうか」という問題は、
プログラムを実際に実行せずにYES/NOを判定することは一般的に不可能ということが証明されている。

つまり、プログラムが停止するかどうかという問いには、プログラムが停止する時にはYESと答えることができるが、停止しない場合は無限に実行を続けるだけになってしまうので、NOと答えられない。

このように、判定機がYes/Noを返すだけではなく、判定きの実行が終わらないことがありえる文法のカテゴリのことを、Turing-recognizableという。
