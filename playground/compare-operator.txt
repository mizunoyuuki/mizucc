この節では、<, <=, >, >=, ==, != を実装する。

これらの比較演算子は、特殊な意味を持っているように見えますが、実際には+や-などと同じように、二つの整数を受け取って一つの整数を返す普通の二項演算子です。+が両辺をたした結果を返すように、例えば == は両辺が同じ場合は、1を違う場合は0を返す。

トークナイザを変更する必要がある。
今までに扱ってきた、記号トークンは長さがどれも１文字で、コードもそれを前提に書かれていた。
== などの比較演算子を扱うためにはコードを一般化する必要がある。
文字の長さをトークンに保存できるように、lenと言うメンバをToken構造体に保存することにする。


比較演算子の追加による新しい文法

比較演算子のサポートをパーサに追加するために、比較演算子を加えた文法がどの表になるのかを考える。
今までに出てきた演算子を優先順位の低い順から高い順に書くと、以下になる

1. == !=
2. <, <=, >, >=
3. + -
4. * /
5. 単項+, -
6. ()


優先順位は生成文法で表現可能で、優先順位の異なる演算子は、別の非終端記号にマップされる。
exprやmulと同様に考えると、比較演算子を加えた新しい文法は以下になる。

expr       =  equality
equality   =  relational ("==" relational | "!=" relational)*
relational =  mul ("<" add | "<=" add | ">" add | ">=" add)*
add        =  mul ("+" mul | "-" mul)*
mul        = unary ("*" unary | "/" unary)*
unary      = ("+" | "-")? primary
primary    = num | "(" expr ")"
