blockを実装している時のまとめ

Node* node {
    ...
    Node **blocks;
    ...
};

この構造体のblocksメンバの扱い、
- どういう構造体なのか
- node->のようにアクセスするときはどうやって書くものなのか。

がいまいちピンと来てなかったため調査

Node *blocks[];

って書き方は、”フレキシブル配列メンバ”って呼ぶらしい、これは構造体の一番したの要素にのみ使うことができて、定義するときに、配列の要素数を変更して割り当てることができる。

Node構造体に動的なblockを持たせる方法。

連結リストを使う。
```
Node node{
  Node *body,
  Node *next,
}
```

body, nextの両方が必要な理由は、blockがネストした時に問題が起きるから。

例えば、 { a; {b; c;} d; } を考える。

```
外側ブロック-> next -> (a) -> next (内側ブロック) -> next (d) -> NULL
                               |
                               --- ??? -> (b) -> next -> (c) -> NULL
```

こんなことになる。

内側ブロックノードのnextは、外側ブロックの連結リストである次の兄弟(d)を指すために使う。
それに加えて、自分の子供(b)も指す必要がある。その時nextが二つの役割を必要として衝突する。

そのためbodyを追加する。
